# Brevium Doctor Appointment Scheduler

## Information On Building Strategy
There were two main requirements that I would like to 
provide more information on so it is clear what I did. 

### AI Usage
The requirement stated 
```
While you are welcome to use AI to help you understand error messages 
or search for documentation, we ultimately want you to be the one who
writes the code, so please do not use AI coding agents or copy and paste 
substantial AI code as part of your submission
```

There is a lot of grey area here, so I just want to be transparent about 
how I used AI. When I first got the email, I immediately began thinking through
how I would like to architect the code and what data structures I would need. 
As part of this planning process, I prompted Claude with my plans and interated a few times
in order to come up with a few data structures that I was confident would work for this project.

During the actual building of the project, I used AI very little, and what 
prompting I did was mainly for cross check on syntax questions.

### Time Constraints
The requirement stated 
```
DO NOT SPEND MORE THAN 3 HOURS ON THIS PROJECT!
```
I received the email for this project while on vacation, and then wasn't able to complete it
for several days after I returned. Because of that, I did have time to plan in my mind (as well as 
some time prompting Claude as I was thinking through my plan as mentioned above) that I wasn't really able to quantify.
This was mostly in spare moments, I never sat down to create an official plan during that time. 

Once I sat down to do the project, I set a timer and completed it in about 2 hours 45 minutes. 
The grey area between mental planning and when I actually sat down to build was unclear, so I wanted
to be transparent about it. When I sat down, I had in mind the data structures that I wanted to use, but not much more.
